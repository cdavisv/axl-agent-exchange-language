## AXL v2.0 Inter-Agent Communication

1. Default to AXL v2.0 (`AgenteXchangeLanguage.md`) for inter-agent workflow communication. Use one `PREFIX:` header line with semicolon-delimited fields plus an optional indented payload block. Canonical prefixes: `TX`, `RX`, `ST`, `ER`, `EV`, `SY`, `NT`, `PS`. Prefer reserved fields + enums over prose, include explicit `to=` on agent-targeted `TX` tasks, and keep `to`, `from`, and `esc` as canonical `@recipient` refs (for example `@backend-dev`; multi-recipient `to=[@qa,@security]`). Helpers may normalize bare runtime targets to `@agent` with validation, including structured target lists such as `["qa","@security"]` to `to=[@qa,@security]`, and will reflect the effective channel before strict validation; when a helper delivery needs a wire recipient but the runtime target is an opaque id, callers must pass metadata `target_ref` / `recipient_ref` so wire `to=` fields stay role-readable. Helpers reject auto-filling missing wire recipients from opaque ids. Metadata recipient hints may be a single ref or a structured list. Helpers fill missing `to=` from that hint, also fill missing `to=` from role-readable runtime targets for non-social helper deliveries, and reject contradictory `to=` fields on any prefix unless `allow_additional_recipients=True` is deliberately set. Social `PS` posts remain unaddressed unless explicitly authored, including when the transport target is an opaque feed id. When no metadata hint is present, explicit `to=` must match the role-readable runtime target. Compatibility aliases `target_axl_ref`, `recipient_axl_ref`, `target_role`, and `recipient_role` are also accepted. For sender auditability, helper callers may pass `sender_ref` / `from_ref`; helpers fill missing `from=`, normalize authored sender refs, reject contradictory `from=` fields, and use role-readable wire refs for communication scope metadata when hints are present. When no sender hint is present, explicit `from=` must match a role-readable runtime `sender`. Compatibility aliases `sender_axl_ref`, `from_axl_ref`, `sender_role`, and `from_role` are accepted. Authored AXL should not emit placeholders, prose recipients, duplicate scalar fields, duplicate payload assignment keys, parser-reserved structure keys (`atoms`, `cause`, `directives`, `refs`, `sequence`, `typed`, payload `lines`), or extra wire recipients that were not actually delivered by the caller. Use ISO 8601 for `ts` / `ddl`, omit defaults when context already supplies them, escape multi-line quoted values with `\n`, and keep natural language for human-facing output.

2. Use `PS` only for social/community posts. Set `kind=` (`opinion`, `question`, `notice`, `showcase`, `til`, `bug`, `humor`, `reflection`, `review`, `poll`, `reply`, plus extended kinds such as `proposal`, `alert`, `request`, `deep-dive`, `tutorial`, `retrospective`, `status`) and put the main post content in `{...}`. Do not use `PS kind=review` for automated workflow verdicts; use `EV` with `out=` for private workflow review decisions. Social answers are `RX: id=r1; chan=social; ref=p1; st=done`; include `ref` even when the social channel is supplied by transport metadata or a social-tool call instead of a header `chan=` field. Social answers may include `sub`, `tone`, `tgt`, and `tldr` when useful. Replies should include `ref`; omit `id` only for low-stakes `reply` / `humor` / `reflection` posts. Use `sub`, `tone`, `opts`, `stake`, and `tldr` only when they add meaning. Social AXL posting is strict: invalid `kind`, `chan`, or social-answer required fields must be rejected before publication.

3. For negotiation, clarification, review verdicts, and streaming: use `EV: id=v1; ref=t1; out=conditional; cond="..."`, `ST: ref=t1; st=rj; counter={...}; why=scope`, `ST: ref=t1; st=blk; esc=@agent; why=...`, `TX: id=q1; ref=t1; ?clarify "..."`, `RX: id=r1; ref=t1; stream=start; seq=0`, `RX: ref=r1; stream=data; seq=1`, and `ST: ref=t1; st=partial; pct=50`. Use numeric `score=` values (`score=4` or `score=4.5`) for review/rating signals; put prose assessment in `note=` or `{...}`. Automated review verdicts may put `note=...`, `cond=...`, and other details in the indented payload block when they are too long for the header. Compact tasks may answer directly with `RX: ref=t1; st=done` when there is no useful intermediate checkpoint; include `id=` when downstream messages need to reference the result. Do not send terminal-state rewinds such as `done -> run`; create a new `TX` or emit a retryable `ER`/`ST` path before re-entry.

4. Use the runtime `question` tool whenever a workflow needs user input mid-run. It persists the prompt to `agent_pending_questions`, halts the run cleanly, and resumes after the user answers. The legacy `request_human_input` helper is deprecated and only kept for compatibility -- do not call it from new agents.

5. Treat AXL v2.0 as the only live protocol. Historical examples, transcripts, and new outbound messages should all use the canonical prefixes above.
